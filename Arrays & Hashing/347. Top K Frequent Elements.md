Given an integer array `nums` and an integer `k`, return the `k` most frequent elements within the array.

The test cases are generated such that the answer is always **unique**.

You may return the output in **any order**.

## Example
```python
Input: nums = [1,2,2,3,3,3], k = 2

Output: [2,3]
```

## Solution
```python
class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        hashmap = {}
        for num in nums:
            if num not in hashmap:
                hashmap[num] = nums.count(num)
        sorted_hashmap = sorted(hashmap, reverse=True, key=lambda x: hashmap[x])
        return list(sorted_hashmap)[:k]
```

We can loop through the numbers list and check if it is in our HashMap. If it is not, then we can add the current number as the key and store the number of times, the count, that the current number appears in the number list. Doing this for all numbers, we can generate a HashMap that contains every number in the numbers list as a key with its corresponding value being the count of that number in the numbers list. Then we can reverse sort the HashMap using the `lamba` function to sort the HashMap by using the values and NOT THE KEYS! That way we can list slicing to return the first `k` most frequent elements within the newly, reverse sorted, array.
