## Problem
Given an integer array `nums` and an integer `k`, return the `k` most frequent elements within the array.

The test cases are generated such that the answer is always **unique**.

You may return the output in **any order**.

## Example
```python
Input: nums = [1,2,2,3,3,3], k = 2

Output: [2,3]
```

## Solution
```python
class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        hashmap = {}
        for num in nums:
            if num in hashmap:
                hashmap[num] += 1
            else:
	            hashmap[num] = 1
        sorted_hashmap = sorted(hashmap.keys(), reverse=True, key=lambda x: hashmap[x])
        return list(sorted_hashmap)[:k]
```

We can loop through the numbers list and check if the current number is in our HashMap.

If it is, then we can add the current number as the key and increase the value, the current number appears in the number list, by 1.

If the current number is not in our HashMap, then we can just set the value to 1.

Doing this for all numbers, we can generate a HashMap that contains every number in the numbers list as a key with its corresponding value being the count of that number in the numbers list. Then we can reverse sort the HashMap using the `lamba` function to sort the HashMap by using the values and NOT THE KEYS! That way we can list slicing to return the first `k` most frequent elements within the newly, reverse sorted, array.
